I will write a paragraph for each methodology of SDLC. There are many of them but I will suggest a few.

My suggestion and reasons are as follows:

  1. Waterfall model: It's an SDLC method known for its simplicity. It's a linear and sequetial approach, which consists of these stages: requirements analysis, design, implementation, testing, deployement, and maintenance. although its not flexible and prefferably used to port an already made software to other platforms, or small projects with fixed and known requirements
  2. Rapid Application model: In this SDLC method, testing is the main component after each stage of devellopement and promotes feedback throughout the devellopement process
  3. Incremental model:It divides the software development process into smaller, manageable increments or iterations. this approach is good to deal with changing requirements and to reduce risks, but requires more planning of each iteration, and which iterations are core features of the program to be implemented first.

Among so many SDLC methodologies the ones that are in the AGILE classification are better. Among AGILE SDLC methods, I have also a few suggestions

  1. Scrum: An iterative agile methods, where a backlog lists all the use cases and features, and the work on each element is divided into sprints. It is a highly flexible methods made for projects with highly changing requirements.
  2. Extreme Programming:Extreme Programming (XP) is an Agile methodology that emphasizes technical excellence and customer satisfaction. It involves frequent releases in short development cycles, which improves productivity and introduces checkpoints where new customer requirements can be adopted. XP practices includs practises like pair programming and more.

I have used CSS in this webpage and there are 3 ways to include CSS in a webpage as follows:

  1. Inline CSS: inside each tag there is a style="..." where the css code is written between the quotation marks
  2. Internal CSS: inside the html file in the head section with the tag "style"
  3. External CSS: with a file that has a format .css linked to the html page (the preffered method i mostly used)

The reasons CSS used in a webpage are:

  1. It makes the webpage look better
  2. It makes the webpage more readable
  3. It makes the webpage more user friendly
  4. It makes the webpage more interactive
